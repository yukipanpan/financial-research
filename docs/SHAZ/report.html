1 · 公司一句话定义

SharonAI 是一家澳洲 Neocloud(新型 AI 云)运营商,作为 NVIDIA Cloud Partner,通过托管在 NEXTDC/Equinix 数据中心的 NVIDIA GPU 集群,以 GPU-as-a-Service + AI PaaS + 高性能存储 的形式、用 take-or-pay(照付不议) 合同向企业/政府/hyperscaler 出租 AI 算力。

公司 2026 年 2 月通过 de-SPAC(壳公司 Roth CH Acquisition 反向并购) 登陆纳斯达克。★ 当前处于 「巨额合同已签、收入尚未启动」 的极早期阶段:FY2025 实际营收仅 $1.57M,却已签下约 $2.2B 的 5 年期合同积压。

3 · 单位约定与常识自检

10 · 关键会计政策深度解析 (铁律 3)

10.1 收入确认(ASC 606)★ Note 2

GPU IaaS 按 over-time(按周)确认——"当设备对客户可用时"。提供 usage-based 与 term-based take-or-pay 两种。历史 Filecoin 挖矿为 point-in-time(WinningPoSt 验证时),已停。

10.2 折旧

PP&E 使用寿命 1–20 年 ★;无利息资本化。GPU 折旧寿命是行业关键变量(同业多用 5–6 年),寿命拉长会美化当期利润——SHAZ 的 1–20 年区间宽泛,需在未来 10-K 关注 GPU 类资产的具体取值。

10.3 可转债公允价值期权(ASC 825)— 最关键 ★

公司对可转债选用公允价值期权,每期重估进损益。这制造了 FY25 −$26.0M、Q1'26 −$70.2M 的非现金损失。机制:公司股权价值上升 → 可转债负债公允价值上升 → 损益录得损失(且不可抵税)。这是"越被看好、账面越亏"的 FVO 悖论。

18 · 我的论点 (含反方)

论点 1:这是一笔"项目融资型"的 pre-revenue 押注,不是一家已验证的云公司

证据:① FY25 营收 $1.57M、Q1 同比下滑;② $2.2B 积压/营收 ~1,400× 远超同业 ~30×;③ 收入靠 NEXTDC 产能 + B300 交付 + 客户转化三重未发生事件。
反方:已是 NVIDIA NCP、已签两个十亿级 take-or-pay、$350M 资金到位,执行兑现即可快速放量。
我的反驳:NCP 与签约降低但未消除执行风险;1,400× 的积压/收入比意味着估值几乎全部押注"按时、按价、按量交付",任一环节滑点都直接证伪。

论点 2:GAAP headline 亏损被非现金项严重扭曲,真实经营亏损要小得多——但这把双刃剑两头都不利

证据:Q1'26 经营亏损仅 −$2.8M,但可转债 FVO −$70.2M 与 TCDC 利得 +$65.9M 对冲后 + 出售税负 → −$19.9M。
反方:剥离非现金项后经营层面亏损可控。
我的反驳:FVO 损失虽非现金,却反映可转债真实的转股稀释/再融资义务($199.4M 列流动);经营亏损小是因为收入还没启动、规模成本尚未发生,不代表盈利能力已验证。

论点 3:治理结构对少数股东不友好

证据:160:1 双层股权、~81.8% 内部人控制、关联方可转债、公众流通极低、审计师频繁更换。
反方:创始人控制在早期高增长公司常见,利于长期决策。
我的反驳:控制权集中 + 关联方借款 + 低透明度(无 RPO/集中度量化)叠加,放大了少数股东的信息与治理风险。
