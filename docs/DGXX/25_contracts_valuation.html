  • 交付：Phase 1（15MW）目标 2026-12-15 RFS；Phase 2（40MW）目标 2027-Q1（额外 25MW 以 Phase 2 融资为条件）
  • 注：DGXX 自有该地产 + 已建专用变电站 + 与 Alabama Power 签供电协议，Phase 1 自筹建设

资金链：ATM → B300 → NeoCloudz → SubQ ⭐

两条 AI 变现路径要分清：① NeoCloudz 自营 GPU 出租（自己买 B300 → 租给 SubQ，资本+折旧在自己表上）；② Cerebras colocation（出租机房空间/电力，按 ASC 842 租赁确认，Phase1 自筹建设）。前者轻客户、短约、自担 GPU 折旧；后者重资产、长约（10 年）、锚定单一大客户。

结论更正：DGXX 不再是"0 签约"。它有 Cerebras（1.1B起/40MW/10年） + SubQ（19.6M / 24 月） 两份已签约 AI 合同，签于 2026-04～05（Q1-2026 报告期后，在 8-K 与 10-Q 后续事件披露）。之前 agent 只读 10-K/10-Q 正文，漏了 8-K Item 1.01 流——这正是 skill「全量 filing 精读」要防的漏。

25.2 同业每兆瓦如何确认收入/成本？用哪种会计准则？

核心机制（三家同业 + DGXX 通用）：收入随『交付的兆瓦』确认，不随『签约』确认。

公司 AI/HPC 收入性质 准则 收入怎么确认 成本怎么确认
CORZ（CoreWeave colocation） 把机房空间/电力租给客户 ASC 842 经营租赁 ★ "lease income" 按租期直线确认，自每个 phase/MW 交付（commence）起 机房建设资本化为 PP&E(ASC 360)，投用后折旧；电力/运维费用化
WULF（HPC 租约） 同上（Core42/Fluidstack） ASC 842 经营租赁 ★ 同上；预付租金（Core42 $90M）递延后摊销 同上
IREN（AI Cloud） GPU 算力即服务 ASC 606 服务 ★ 按交付/验收批次确认；RPO 已披露但交付前为 nil GPU 资本化折旧 + 运营费用化
DGXX（Cerebras） colocation + 主服务 大概率 ASC 842（独家使用+按 kW 月费）或部分 606（合同太新，公司尚未披露口径，待查） 预计同 CORZ/WULF：交付起按期确认；预付递延 同上（建设资本化折旧）

关键推论（直接回答"市值反映已交付还是已签约"）：

1. 损益表只反映『已交付/已投用的 MW』——签约但未交付的容量不进收入。
2. 已签未交付的部分 = 附注里的『未来经营租赁应收款 / RPO』（backlog），在表外。
   例：CORZ 披露 ~$5.8B『尚未 commence 的未来不可撤销经营租赁应收款』(12 年期)。
3. 因此：当期收入/EBITDA 远小于市值能解释的水平——市值必然在给 backlog + 未来 ramp 定价。

读这张表的三个结论：

(1) 市值绝不只反映已交付 MW。 若市值只给已通电 MW 定价，WULF 的 $255M/已通电MW 是荒谬的——说明市场在给 522MW 已签约 + 未来扩张定价，而非 60MW 已通电。三家同业皆如此：市值 = 已签 backlog 的折现 + 成长期权，远超已交付部分。

(2) 按"每已签约兆瓦"，四家收敛到 ~16–29M/MW。 * *DGXX签下Cerebras后 ≈ **16M/MW，与 CORZ 持平、略低于 WULF——意味着 Cerebras 大单基本抹平了 deck 之前宣称的"矿企折价"，市场已把 DGXX 朝同业的"每签约兆瓦"水平重估。（注：DGXX 的 deck 曾称 HPC ~$12.5M/MW，与此区间同量级。）

(3) 按"每已签 backlog 美元"，DGXX(~0.6x) < CORZ(~1.0x) < IREN(~1.85x)。 表面看 DGXX 最"便宜"，但这恰恰反映确定性的差异（见下），不能简单当低估。

25.4 那么——高估 / 低估 / 合理 pricing？（分析框架，非投资建议）

⚠️ 本节是研究性框架，不构成投资建议，不下买卖结论。是高是低取决于下面这些"你必须自己判断的假设"。

为什么不能只看每兆瓦数字下结论——决定 over/under/fair 的是 backlog 的"含金量"：

维度 DGXX IREN WULF CORZ
客户信用 Cerebras（资金雄厚但未上市 AI 芯片公司） Microsoft + NVIDIA（蓝筹） Fluidstack（Google 背书/信用增强） CoreWeave（IG 评级 SPV）
backlog 确定性 较低（单一客户、Phase1 未交付） 中（RPO 仍 nil，需交付） 较高（Google 背书） 较高（12 年、IG SPV）
交付进度 最早期（2026-12 才 Phase1） 爬坡中 60/522MW 已通电 225/590MW 已计费
杠杆 零债务 $3.7B $5.2B 2.1B(→5.4B)
规模 最小 最大 中 中

框架式判断（"如果……则……"，不是结论）：

  • 若你认为已签 backlog 大概率按期转成收入 → 当前市值更像在给"已签约 + 合理成长"定价，处于接近合理 pricing；其中按 EV/已签约MW，四家相近，市场定价相对一致。
  • 若你认为市场还额外计入了"尚未签约的未来合同"（成长期权）→ 则对交付/签约最不确定的玩家（早期、单客户）存在透支风险。
  • DGXX 的特殊点：EV/backlog 最低（~0.6x）但确定性最低（单一未上市客户 + 未交付 + 规模最小）。"便宜"与"高风险"是同一枚硬币——它到底是低估还是合理，取决于你给 Cerebras 这单的履约概率和Cerebras 自身信用打几折。
  • 同样，IREN 的 EV/backlog 最高（~1.85x）也不等于高估——它的客户（微软/英伟达）信用最好、合同最大，市场愿意给溢价。

一句话总结（研究观点，非建议）： 这几家的市值都不是只反映已交付兆瓦，而是在给『已签约 backlog 的折现 + 成长期权』定价。按"每已签约兆瓦"看，四家收敛在 ~$16–29M/MW、相对一致，更像"正常 pricing"而非普遍高/低估；真正的分歧不在"每兆瓦多少钱"，而在你给每家的 backlog 履约概率和客户信用打几折——这是判断个股高低估的真正变量，需你自行假设。我不替你下买卖结论。
